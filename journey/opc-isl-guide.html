01 설치
1

인스톨러 다운로드

NAS 배포 폴더에서 OPC-ISL-x.x.x-setup.exe를 로컬 PC로 복사합니다.

2

설치 실행

복사한 인스톨러를 실행합니다. UAC 요청 시 "예"를 클릭하세요.

3

WebView2 런타임 확인

Windows 11은 기본 탑재. Windows 10 미설치 시 인스톨러가 자동으로 다운로드합니다.

4

초기 실행 — 매퍼 파일 지정

최초 실행 시 isl_mapper.xlsx 경로를 지정하고 "설정 완료 및 시작"을 클릭합니다.

02 사용법 — 2단계 워크플로우
Step A ISL 자동 생성
→ ISL_TagList_*_a_*.xlsx
OPC 원본 파일 XLSX 또는 CSV 필수
발전소명 파일명에서 자동 감지 자동
발전소 타입 PV / BESS / Hybrid — 태그명 기반 자동
OPC 형식 KepServer / SCADA IO 자동
"ISL 생성" 버튼 클릭 → 원본 파일과 동일 폴더에 결과 생성
ISL 파일 전달
Step B HEIS Import 파일 생성
→ HEIS_Import_*.xlsx
ISL 파일 Step A 결과 파일 필수
HEIS 템플릿 Row4=헤더, Row5=기본값 형식 필수
SERVER_ID / NAME HEIS 서버 식별자 및 이름
AGENT_ID / NAME HEIS 에이전트 식별자 및 이름
ConnectionId OPC 연결 ID
PollingGroup A/D 아날로그 / 디지털 폴링 그룹명
NamespaceIndex 0 ~ 99 범위
"Import 파일 생성" 클릭 → HEIS_Import_*.xlsx 생성
03 초기 설정
ISL 매핑 룰 파일
최초 실행 시 isl_mapper.xlsx 경로 지정. 설정은 영구 저장됩니다.
설정 폴더 위치
%USERPROFILE%\.hanwha\opc\
화면 초기화
입력 필드만 초기화. 매퍼 파일 경로 설정은 유지됩니다.
04 주의사항
⚠ NAS에서 직접 실행 금지
네트워크 경로(UNC)에서 exe를 직접 실행하면 사이드카 바이너리 로딩이 실패합니다. 반드시 로컬 PC에 설치 후 실행하세요.
⚠ HEIS 템플릿 형식 준수
Import 생성 시 템플릿의 4행 = 헤더, 5행 = 기본값 형식이 반드시 지켜져야 합니다.
⚠ OPC 형식 자동 감지 실패 시
감지 실패 시 기본값 KepServer로 처리됩니다. 결과 파일 확인 후 필요 시 수동 지정하세요.
✔ 경고 발생 시 로그 자동 표시
처리 중 경고가 발생하면 앱 하단에 로그가 자동 표시됩니다. 상세 내용은 로그 파일을 확인하세요.
05 로그 위치

문제 발생 시 아래 경로의 로그 파일을 확인하거나 담당자에게 전달하세요.

%USERPROFILE%\.hanwha\opc\logs\api.log

앱 내 "설정 폴더 열기" 버튼으로 해당 디렉토리에 바로 접근할 수 있습니다.
