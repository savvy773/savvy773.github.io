🏠 HEIS Tool — 통합 데이터 관리 플랫폼

HEIS Tool은 태양광 발전소 데이터를 수집·변환·검증하는 3개의 핵심 도구와 실시간 서버 모니터로 구성된 통합 플랫폼입니다.

☁️
Backfill Tool
데이터 이관 · 추출 · 적재

서버(InfluxDB) 데이터를 추출하거나 로컬 파일을 서버에 적재하는 전체 파이프라인을 제공합니다.

  • CSV 전처리 및 기간 필터링 (Trim)
  • Line Protocol 형식 변환 + Fuzzy 매칭 (Convert)
  • 서버 부하 자동 감지 · 속도 조절 (Extract)
  • 대용량 파일 일괄 적재 (Load)

Tag Generator

  • 설비 기준 정보 자동 추출 (Step 2)
  • 파라미터·단위·스케일 자동 매핑 (Step 3)
  • 알람 등급 자동 분류 (Step 4~5)

v1.5.2 기준 — 각 레이어의 핵심 기술만 명시합니다.

🖥️ Runtime Python 3.14+  ·  PySide6 / Qt6 (데스크톱 UI)  ·  QML (설정 페이지)
🗄️ Database MySQL/MariaDB (homdb, 매핑·태그 기준정보)  ·  InfluxDB 1.x (시계열 센서 데이터)
SSH → mysql -N -B TSV 스트림  ·  MappingRepository 세션 캐시  ·  Excel fallback
🔒 Security keyring (Win Credential Manager / macOS Keychain) → fallback AES-256-GCM + PBKDF2 (320k iter)
패스프레이즈: MAC+hostname SHA-256 머신 파생  ·  MySQL full-escape (백슬래시·NUL·CR·LF 포함)  ·  InfluxQL tag_filter 화이트리스트  ·  DEV/PRD 크레덴셜 분리
📡 SSH OpenSSH (System32 > shutil.which, Win/Mac/Linux 자동 탐지)  ·  ProxyJump 3-hop 터널
재시도 + 지수 백오프 (최대 2회)  ·  PEM chmod 400 / icacls  ·  config chmod 600  ·  atexit 자동 정리
⚙️ 설정·빌드 단일 버전 소스: heis_main.py __version__ → pyproject.toml / __init__.py / HEIS_guide.html 자동 동기화
OS별 기본 경로 자동 선택 (Win/macOS/XDG)  ·  backfill_config 상수 — IP·User 하드코딩 전면 제거  ·  Briefcase 빌드
🗺️ 매핑 DB-first: TB_TAG_STDR_INFO ⋈ TB_SCADA_TAG_INFO  ·  tb_scale_info FORML 수식 eval  ·  SP_GET_HEIS_TAG_INFO 저장 프로시저
SSH 불가 시 _mapping.xlsx 자동 fallback  ·  gzip 압축 업로드로 Excel → MySQL 일괄 임포트

🛡️ PEM 파일 권한 설정 앱 실행 시 자동

HEIS Tool 실행 시 자동 적용됩니다. 수동 실행이 필요한 경우에만 아래 명령을 사용하세요.

🪟 Windows PowerShell

$pem = "$env:USERPROFILE\Documents\_tunnel\pem"
Get-ChildItem "$pem\*.pem" | ForEach-Object {
    icacls $_.FullName /inheritance:r /grant:r "${env:USERNAME}:R"
    Write-Host "✅ $($_.Name)"
}

🐧 Linux / macOS

chmod 400 ~/Documents/_tunnel/pem/*.pem

🛠️ Trim — CSV 전처리

대용량 원본 CSV 파일에서 설정한 날짜 범위만 잘라내어 trim/ 하위 폴더에 저장합니다. 또는 여러 CSV를 하나로 병합합니다.

주요 기능
  • 날짜 범위 필터링 — 수백만 행도 빠르게 처리
  • UTC ↔ KST 시간대 자동 보정
  • Merge — 복수 CSV를 단일 파일로 합치기

02

🚀 Convert — Line Protocol 변환

SCADA CSV를 InfluxDB 전용 Line Protocol(.lp) 형식으로 변환합니다. 엑셀 매핑 파일로 컬럼명·스케일을 자동 처리합니다.

주요 옵션
  • 최대 파일 크기 제한 (MB 단위 분할)
  • Host 태그 모드: default / custom / no_host
  • 파일명 유효성 검증 (Fuzzy Matching 포함)
  • 기간 필터링, 단일 LP 병합 옵션
입력 / 출력
📥 정제된 CSV + 엑셀 매핑 파일
📤 .lp 파일 (InfluxDB 적재용)
태그명 불일치도 Fuzzy 자동 매칭
03

📡 Extract — 서버 → 로컬

DEV 또는 PRD InfluxDB에서 데이터를 추출해 로컬 .lp 파일로 저장합니다. 서버 부하를 실시간 감지해 자동 스로틀링합니다.

주요 옵션
  • 추출 환경 선택 (DEV / PRD)
  • 날짜 범위 + 추출 간격 (1h / 1d 등) 설정
  • 포함/제외 태그 필터, Measurement 선택
  • gzip 압축 여부, 태그 제외 패턴
스로틀링
서버 iowait > 15% 시 자동 일시 중지
로그에서 [THROTTLE] 메시지 확인
중지 후 자동 재개

  • 동기화 지연(초) — 파일 간 전송 간격 조절
  • 로컬 정리 / 원격 정리 자동 삭제 옵션

💬 로그에 [THROTTLE] iowait=XX% — pausing 메시지가 표시됩니다.
🔀 스마트 노드 자동 선택 PRD 전용

PRD 환경 작업 시 Node 140(PRD)과 Node 170(PRD2) 중 현재 쓰기 요청을 처리 중인 활성 노드를 자동으로 감지하여 접속합니다. 덜 바쁜 서버로 자동 분산됩니다.

💬 Telegraf가 HTTP 204 응답을 반환하는 노드가 활성 노드로 판별됩니다.
🔁 SSH 터널 자동 갱신 5분 주기

서버 접속은 SSH 보안 터널을 통해 이루어집니다. 장시간 실행 시 터널이 끊길 수 있어 5분마다 자동으로 갱신합니다. 작업 중 연결이 끊기는 상황을 방지합니다.

💬 별도 조작 없이 백그라운드에서 자동 처리됩니다.
🔄 자동 재연결 지수 백오프

네트워크 불안정 등으로 연결이 끊기면 모니터에 [OFFLINE]이 표시되고 5초 → 10초 → 30초 간격으로 자동 재연결을 시도합니다. 자동 복구되므로 별도 조작이 필요 없습니다.

💬 연결 복구 시 모니터 수치가 자동으로 업데이트됩니다.

🖱️ 모니터 UI 사용 방법

탭 전환 (INFLUX / BASTION)

사이드바 상단의 탭을 클릭하면 InfluxDB 상세 정보 또는 Bastion 서버 정보로 전환됩니다.

ℹ️
Load 5 경고선 안내

가로 바 그래프의 분홍색 점선이 Load 5 기준선입니다. PRD 서버가 이 수준을 초과하면 작업 속도가 자동으로 줄어들며, 가능하면 작업을 잠시 대기하거나 서버 부하가 낮은 시간대에 진행하세요.
